🔑:`tmux`（Terminal Multiplexer）是一款强大的终端复用工具，主要用于在单个终端窗口中高效管理多个终端会话。它特别适合需要同时处理多个任务、远程工作或希望提升终端使用效率的用户。--- 核心功能：1. 会话管理： - 会话持久化：即使断开 SSH 连接或关闭终端窗口，会话中的任务仍会在后台运行，重新连接后可恢复工作环境。 - 多会话支持：可同时管理多个独立会话（如开发、调试、运维），方便任务隔离。2. 窗口（Window）与窗格（Pane）： - 多窗口：在一个终端中创建多个“标签页”（类似浏览器标签），每个窗口运行独立任务。 - 分割窗格：将窗口垂直或水平分割为多个窗格，同时操作不同任务（如左侧编码、右侧查看日志）。3. 协作共享： - 允许多个用户同时连接同一会话，适合结对编程或远程协助。4. 高度可定制： - 通过配置文件（`~/.tmux.conf`）自定义快捷键、主题、状态栏等。 - 支持插件扩展（如 `Tmux Plugin Manager`），增强功能。--- 典型使用场景：- 远程服务器工作：通过 SSH 连接服务器时，即使网络中断，任务也不会终止。- 多任务并行：例如同时监控日志、运行代码、操作数据库等。- 长耗时任务：后台运行编译、下载等任务，释放当前终端。- 开发环境配置：保存常用工作布局（如 IDE、终端、日志监控的窗格组合），快速恢复。--- 基础操作示例：- 启动会话：终端输入 `tmux`。- 快捷键前缀：默认按 `Ctrl + b` 后接命令： - 水平分割窗格：`Ctrl + b` → `%` - 垂直分割窗格：`Ctrl + b` → `"` - 切换窗格：`Ctrl + b` → 方向键 - 新建窗口：`Ctrl + b` → `c` - 脱离会话：`Ctrl + b` → `d` - 恢复会话：`tmux attach -t <会话名>`。--- 与 GNU Screen 的区别：`tmux` 是更现代的替代品，提供更直观的窗格分割、更好的配置支持以及更活跃的社区。功能类似，但 `tmux` 的快捷键和扩展性通常更受青睐。---如果你经常使用命令行，尤其是涉及多任务或远程工作，`tmux` 可以显著提升效率和可靠性。
